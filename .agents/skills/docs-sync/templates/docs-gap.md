## How to use this file

**Adding a gap:**
When a milestone closes and a feature's `user_visible_behavior` is not
reflected in any docs file, add a row to the Undocumented Features table.

**Clearing a gap:**
When documentation is written, move the row to Recently Cleared and fill
in the "Documented in" column with the file path or section name.

**Priority guide:**
- `high` — user-facing feature with no documentation at all
- `med` — documented but outdated or incomplete
- `low` — internal-facing or advanced feature with low discoverability
