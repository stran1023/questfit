# QuestFit Screenshots

These screenshots are project-owned application captures generated by the desktop Chrome Playwright journey with `CAPTURE_VISUAL_EVIDENCE=1`.

- `plan-your-route.png` shows a validated mobility briefing with the route, plan-fit rationale, and AI-to-action pipeline.
- `ash-titan-battle.png` shows the active Phaser boss encounter after deterministic synthetic movement events advance the mission.
- `mission-results.png` shows deterministic completion metrics and fact-locked coaching after all verified objectives complete.

The capture journey intentionally denies camera access and initializes a validated local mission fixture. It does not fabricate product UI, calculated metrics, or Phaser state: the normal mission controller processes the same typed movement-event boundary used by on-device pose recognition. Real-camera behavior remains a separate physical demo requirement.

Regenerate the source captures with:

```bash
CAPTURE_VISUAL_EVIDENCE=1 npx playwright test tests/e2e/hardening.spec.ts \
  --project=desktop-chrome \
  --grep 'goal-aware planning|mission world'
```
